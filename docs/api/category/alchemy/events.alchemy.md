# События Алхимии

## EVENT_ALCHEMY_CANCELED

Передает статус завершения или прерывания процесса алхимии.

### Описание

```lua
EVENT_ALCHEMY_CANCELED
```

Событие приходит при завершении или принудительном прерывании использования умения алхимии. Параметр `isSuccess` определяет итог процесса: значение `false` фиксирует прерывание (например, из-за перемещения, прыжка или гибели аватара), значение `true` подтверждает штатное завершение действия.

### Список параметров

- **`isSuccess`** (`boolean`) - Флаг успешности завершения действия. Принимает значение `false`, если процесс был прерван, и `true`, если действие завершилось в штатном режиме.

### Возвращаемые значения

Обработка события производится в зарегистрированном обработчике.

### Примеры

#### Обработка прерывания алхимии

```lua
function OnAlchemyCanceled( eventData: table )
    if not eventData.isSuccess then
        LogInfo( "Алхимия прервана или окно закрыто." )
    else
        LogInfo( "Алхимия завершена успешно." )
    end
end

common.RegisterEventHandler( OnAlchemyCanceled, "EVENT_ALCHEMY_CANCELED" )
```

## EVENT_ALCHEMY_ITEM_PLACED

Уведомляет об успешном размещении или извлечении компонента в слот ступки.

### Описание

```lua
EVENT_ALCHEMY_ITEM_PLACED
```

Событие передается после успешного выполнения операции помещения или извлечения алхимического компонента в указанный слот ступки (посредством вызова `avatar.PutAlchemyItemToSlot`).

### Список параметров

- **`slot`** (`number`) - Индекс слота ступки, над которым было произведено действие.

- **`placed`** (`boolean`) - Флаг действия. Значение `true` указывает на успешное размещение компонента, `false` - на его извлечение.

### Возвращаемые значения

Обработка события производится в зарегистрированном обработчике.

### Примеры

#### Отслеживание заполнения слотов

```lua
function OnAlchemyItemPlaced( eventData )
    local action = eventData.placed and "положен в" or "извлечен из"
    LogInfo( "Компонент " .. action .. " слот " .. eventData.slot )
end

common.RegisterEventHandler( OnAlchemyItemPlaced, "EVENT_ALCHEMY_ITEM_PLACED" )
```

## EVENT_ALCHEMY_NOT_AVAILABLE_RESOURCES

Информирует о нехватке предметов в инвентаре для добавления в ступку.

### Описание

```lua
EVENT_ALCHEMY_NOT_AVAILABLE_RESOURCES
```

Событие передается при неудачной попытке добавления компонента в ступку по причине отсутствия необходимого количества таких предметов в инвентаре персонажа.

### Список параметров

- **`itemId`** (`ObjectId`) - Идентификатор предмета, который предпринималась попытка поместить в ступку.

- **`slot`** (`number`) - Индекс слота ступки (`0` - первый слот), в который планировалось добавить предмет.

### Возвращаемые значения

Обработка события производится в зарегистрированном обработчике.

## EVENT_ALCHEMY_STARTED

Уведомляет об активации умения алхимии и готовности к загрузке ингредиентов.

### Описание

```lua
EVENT_ALCHEMY_STARTED
```

Событие приходит при начале действия умения алхимии после использования соответствующего алхимического инструмента. С этого момента открывается возможность размещения ингредиентов в слоты ступки посредством вызова `avatar.PutAlchemyItemToSlot`.

### Список параметров

Параметры отсутствуют.

### Возвращаемые значения

Обработка события производится в зарегистрированном обработчике.

### Примеры

#### Инициализация состояния при старте алхимии

```lua
function OnAlchemyStarted()
    LogInfo( "Окно алхимии открыто." )
end

common.RegisterEventHandler( OnAlchemyStarted, "EVENT_ALCHEMY_STARTED" )
```
