# auction.Search

Запрашивает поиск аукционов по заданным параметрам, применяя фильтр, сортировку и пагинацию.

## Описание

```lua
auction.Search( filter, orderField, asc, page )
```

Функция посылает процесс поиска аукционов на основе заданных параметров. Критерии поиска определяются в таблице-фильтре, дополнительно задаются поле для сортировки, направление сортировки и номер запрашиваемой страницы результатов.

Поле сортировки указывается с использованием перечисления `AUCTION_ORDERFIELD_...`. Отдельные поля фильтра (например, `itemClass`, `rarityMin`, `rarityMax`) требуют указания строкового псевдонима соответствующего ресурса. Валидность маски для поиска по имени предварительно проверяется функцией `auction.IsSearchNameValid`.

Список найденных аукционов передается на клиент постранично. Функция `auction.GetAuctions()` возвращает список только для указанной при поиске страницы (доступной через `auction.GetAuctionsPage()`), а общее количество страниц можно получить с помощью `auction.GetAuctionsPageCount()`.

::: warning Замечание
Запуск нового поиска невозможен до завершения предыдущего. Статус выполнения проверяется функцией `auction.IsSearchInProgress()`.
Результат выполнения возвращается через событие `EVENT_AUCTION_SEARCH_RESULT`.
:::

## Список параметров

- **`filter`** (`table`):
  - `name` (`WString` | `nil`) — маска для поиска по имени.
  - `itemClass` (`string` | `nil`) — псевдоним класса предмета.
  - `dressSlot` (`number` | `nil`) — слот одежды (перечисление `DRESS_SLOT_...`).
  - `rarityMin` (`string` | `nil`) — псевдоним минимального качества предмета.
  - `rarityMax` (`string` | `nil`) — псевдоним максимального качества предмета.
  - `levelMin` (`number` | `nil`) — минимальный уровень предмета.
  - `levelMax` (`number` | `nil`) — максимальный уровень предмета.
  - `bidMin` (`number` | `nil`) — минимальная последняя ставка.
  - `bidMax` (`number` | `nil`) — максимальная последняя ставка.
  - `buyoutMin` (`number` | `nil`) — минимальная цена выкупа.
  - `buyoutMax` (`number` | `nil`) — максимальная цена выкупа.
  - `onlyMyAuctions` (`boolean` | `nil`) — флаг отображения только аукционов, созданных главным игроком.
  - `onlyAuctionsWithMyBids` (`boolean` | `nil`) — флаг отображения только аукционов с последней ставкой от главного игрока.
  - `rootCategory` (`ItemCategoryId` | `nil`) — идентификатор корневой категории.
  - `childCategory` (`ItemCategoryId` | `nil`) — идентификатор терминальной категории.

- **`orderField`** (`number`)
Поле сортировки результатов. Задается значением из перечисления `AUCTION_ORDERFIELD_...`. 

- **`asc`** (`boolean`)
Направление сортировки. Если `true`, сортировка выполняется по возрастанию; если `false` — по убыванию.

- **`page`** (`number`)
Номер страницы результатов для отображения, начиная с 1.

## Возвращаемые значения

Возвращаемое значение отсутствует. Результат поиска передается через событие `EVENT_AUCTION_SEARCH_RESULT`.

## Примеры

### Поиск предметов определенного уровня с сортировкой

```lua
if not auction.IsSearchInProgress() then
    local filter = {}
    filter.levelMin = 10
    filter.levelMax = 12
    auction.Search( filter, AUCTION_ORDERFIELD_LEVEL, false, 1 )
end
```

::: info Описание примера
В примере выполняется проверка, что в данный момент не идет другой процесс поиска (`auction.IsSearchInProgress()`). Затем формируется таблица `filter` с ограничением уровня предмета от 10 до 12.
Вызов `auction.Search` начинает поиск на первой странице (`1`), с сортировкой по уровню (`AUCTION_ORDERFIELD_LEVEL`) по убыванию (`false`).
Сами результаты будут доступны в событии `EVENT_AUCTION_SEARCH_RESULT`.
:::
