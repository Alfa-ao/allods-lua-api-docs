# avatar.AcceptQuest
Принимает задание у NPC, находящегося в текущей цели аватара.

## Описание
```lua
avatar.AcceptQuest( questId: QuestId )
```
Функция инициирует процесс принятия задания от NPC, который в данный момент выбран главным персонажем в качестве текущей цели. Для успешного выполнения операции NPC обязан находиться в фокусе цели аватара, так как контекст взаимодействия жестко привязан к выбранному юниту на стороне сервера. 

Вызов функции допускается исключительно после предварительного запроса списка доступных взаимодействий с помощью функции `avatar.RequestInteractions` (в исходной документации упоминается как `FunctionAvatarRequestInteractions`). Сервер должен обработать запрос и передать клиенту актуальный список диалогов и заданий. Попытка принять задание, которого нет в полученном списке, или вызов функции без предварительного запроса взаимодействий приведет к игнорированию команды.

## Список параметров
- **`questId`** (`QuestId`)
Уникальный идентификатор задания, которое необходимо принять. Идентификатор должен строго соответствовать одному из доступных заданий, полученных в списке взаимодействий текущего NPC.

## Возвращаемые значения
Возвращаемое значение отсутствует.

## Примеры
### Принятие задания у текущей цели
```lua
local targetId = avatar.GetTargetId()

if targetId then
    -- Запрос списка взаимодействий с NPC
    avatar.RequestInteractions( targetId )
    
    -- Идентификатор задания, полученный из списка доступных взаимодействий
    local questId = 12345 
    
    -- Непосредственное принятие задания
    avatar.AcceptQuest( questId )
end
```
::: info Описание примера
В примере сначала извлекается идентификатор текущей цели аватара. При наличии цели инициируется запрос взаимодействий через `avatar.RequestInteractions`. После получения актуального списка заданий от сервера (в реальном сценарии извлечение `questId` происходит в обработчике события `EVENT_INTERACTIONS`) вызывается функция `avatar.AcceptQuest` для принятия конкретного задания.
:::
