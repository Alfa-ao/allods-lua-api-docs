# avatar.GetAvailableItemQuests
Возвращает список заданий, которые предмет из инвентаря может выдать персонажу.

## Описание
```lua
avatar.GetAvailableItemQuests( itemId: ObjectId ): table | nil
```
Функция возвращает список идентификаторов квестов, которые могут быть выданы предметом, находящимся в инвентаре персонажа. Для получения актуального списка заданий необходимо предварительно инициировать запрос данных с сервера с помощью функции `avatar.RequestQuestsForItem`.

::: warning Замечание
Функция возвращает результаты только после успешного выполнения запроса `avatar.RequestQuestsForItem`. Повторные вызовы до получения ответа от сервера могут вернуть устаревшие или пустые данные.
:::

## Список параметров
- **`itemId`** (`ObjectId`)
Уникальный идентификатор предмета. Предмет обязательно должен находиться в слоте одного из контейнеров персонажа (основная сумка, переполненная сумка или банковская ячейка).

## Возвращаемые значения
Возвращает таблицу (`table`), содержащую идентификаторы квестов (`QuestId`). Индексация элементов в таблице начинается с единицы. Если предмет не может выдать заданий или данные не были запрошены, возвращается пустая таблица или `nil`.

## Примеры
### Получение списка доступных заданий от предмета
```lua
local itemId = 12345 -- Идентификатор предмета из инвентаря

-- Предварительный запрос данных с сервера
avatar.RequestQuestsForItem( itemId )

-- Получение списка доступных заданий
local availableQuestList = avatar.GetAvailableItemQuests( itemId )

if availableQuestList then
    for i = 1, #availableQuestList do
        local questId = availableQuestList[ i ]
        LogInfo( "Доступен квест: ", questId )
    end
end
```
::: info Описание примера
В примере сначала выполняется запрос актуальных данных о квестах для предмета с идентификатором `12345` через `avatar.RequestQuestsForItem`. Затем с помощью `avatar.GetAvailableItemQuests` извлекается таблица идентификаторов доступных заданий. Если таблица успешно получена, происходит перебор всех элементов (индексация начинается с 1) и вывод идентификатора каждого квеста в информационный лог.
:::
