# avatar.GetBaseTalentInfo
Возвращает информацию о таланте из базовой таблицы талантов.

## Описание

```lua
avatar.GetBaseTalentInfo( layer: number, line: number ): table | nil
```

Функция возвращает детальную информацию о таланте, расположенном в базовой таблице талантов по указанным координатам строки и столбца. Возвращаемая таблица содержит данные о текущем состоянии таланта, его рангах, требованиях для изучения, а также информацию о родительских талантах и подменах. Если таланта в указанной ячейке не существует, функция возвращает `nil`.

::: warning Замечание
В режиме `Край мира` возвращаемая таблица дополнительно содержит поле `relatedSpells`, представляющее собой список активных способностей, на которые влияет данный талант.
:::

## Список параметров

- **`layer`** (`number`)
Номер строки в базовой таблице талантов.

- **`line`** (`number`)
Номер столбца в базовой таблице талантов.

## Возвращаемые значения
(HELP) (MEGA)
Возвращает `table` или `nil`.
Если талант существует, возвращается таблица со следующими полями:
- **`canUpdate`** (`boolean`)
Возможность изучить или обновить данный талант.
- **`isEmpty`** (`boolean`)
Является ли клетка пустой.
- **`isLearned`** (`boolean`)
Выучена ли клетка таланта.
- **`priorityType`** (`string`)
Приоритет таланта. Значения соответствуют перечислению `ENUM_PriorityType_*`:
  - `ENUM_PriorityType_Default` — обычный талант.
  - `ENUM_PriorityType_Recommended` — рекомендуемый талант (серебряная звездочка), не изучается автоматически при использовании ковчега.
  - `ENUM_PriorityType_Suggested` — рекомендуемый талант (золотая звездочка), изучается автоматически при использовании ковчега.
- **`requiredSpentTP`** (`number`)
Необходимое количество вложенных очков талантов для изучения данного таланта.
- **`currentRank`** (`number`)
Текущий выученный ранг таланта (индекс в списке `ranks`, начиная с 1). Если талант не изучен, возвращается `0`.
- **`nextRank`** (`number` | `nil`)
Следующий для изучения ранг таланта (индекс в списке `ranks`, начиная с 1). Возвращается `nil`, если следующий ранг нельзя изучить.
- **`replacement`** (`table` | `nil`)
Подменяющие талант данные.
  - `text` (`WString`) — подменяющее описание.
  - `image` (`TextureId`) — идентификатор текстуры для иконки (поле доступно только для UI).
- **`ranks`** (`table`)
Индексированная с 1 таблица с информацией о рангах таланта. Каждый ранг описывается таблицей:
  - `rank` (`number`) — ранг (начиная с 1).
  - `name` (`WString`) — имя.
  - `desc` (`WString`) — описание.
  - `spellId` (`SpellId` | `nil`) — идентификатор ресурса заклинания (если есть, иначе `nil`).
  - `abilityId` (`AbilityId` | `nil`) — идентификатор ресурса умения (если есть, иначе `nil`).
  - `image` (`TextureId`) — идентификатор текстуры для иконки (поле доступно только для UI).
- **`requiredUnlocks`** (`table`)
Список идентификаторов (`UnlockId`) возможностей, которые должен иметь игрок, чтобы выучить талант.
- **`parentTalentInfo`** (`table` | `nil`)
Информация о родительском таланте, если он существует.
  - `spellId` (`SpellId` | `nil`) — идентификатор ресурса заклинания, требуемого для изучения этого таланта.
  - `abilityId` (`AbilityId` | `nil`) — идентификатор ресурса умения, требуемого для изучения этого таланта.
  - `layer` (`number`) — номер строки.
  - `line` (`number`) — номер столбца.
- **`relatedSpells`** (`table` | `nil`)
Доступно только в режиме «Край мира». Список идентификаторов (`SpellId`) активных способностей, на которые влияет данный талант. Индексация начинается с 1.
## Примеры
### Получение информации о таланте
```lua
local layer = 1
local line = 2
local talentInfo = avatar.GetBaseTalentInfo( layer, line )

if talentInfo and not talentInfo.isEmpty then
    LogInfo( "Талант найден. Текущий ранг: ", talentInfo.currentRank )
    if talentInfo.isLearned then
        LogInfo( "Талант изучен." )
    end
else
    LogInfo( "Ячейка пуста или талант не существует." )
end
```
::: info Описание примера
В примере запрашивается информация о таланте, расположенном на пересечении первой строки и второго столбца. Если таблица успешно получена и ячейка не является пустой, в лог выводится текущий ранг таланта и проверяется флаг его изучения. В противном случае выводится сообщение об отсутствии таланта.
:::
