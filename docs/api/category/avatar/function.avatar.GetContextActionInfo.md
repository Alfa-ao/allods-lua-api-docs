# avatar.GetContextActionInfo
Возвращает детальную информацию о контекстном действии аватара.

## Описание
```lua
avatar.GetContextActionInfo( id: ObjectId ): table | nil
```
Извлекает и возвращает полную информацию о контекстном действии из списка действий аватара. Контекстное действие может присутствовать в списке, но быть недоступным для использования (например, цель находится вне зоны действия или в другой полусфере игрока). Поле `enabled` принимает значение `true`, если действие доступно, и `false`, если оно недоступно. При изменении значения данного поля клиенту передается событие `EVENT_CONTEXT_ACTIONS_CHANGED`.

Возвращает таблицу с данными о действии или `nil`, если действие с указанным идентификатором отсутствует.

::: tip Совет
Для повышения производительности рекомендуется использовать более оптимизированные варианты получения данных: `avatar.GetContextActionShortInfo( id )` для получения краткой информации и `avatar.GetContextActionCooldown( id )` для получения данных о времени восстановления.
:::

## Список параметров
- **`id`** (`ObjectId`)
Уникальный идентификатор контекстного действия.

## Возвращаемые значения
Возвращает таблицу (`table`) или `nil`.

Если действие найдено, возвращается таблица со следующими полями:
- **`id`** (`ObjectId`) — идентификатор контекстного действия.
- **`priority`** (`number`) — приоритет действия.
- **`name`** (`WString`) — название действия.
- **`description`** (`ValuedText` | `nil`) — описание с подставленными текущими значениями параметров.
- **`image`** (`TextureId`) — идентификатор текстуры для иконки (реализовано для UI).
- **`enabled`** (`boolean`) — индикатор доступности действия.
- **`sysType`** (`string`) — строковый идентификатор типа контекстного действия.
- **`objectId`** (`ObjectId` | `nil`) — идентификатор объекта, к которому применяется действие.
- **`spellId`** (`SpellId` | `nil`) — идентификатор ресурса заклинания.
- **`itemId`** (`ObjectId` | `nil`) — идентификатор предмета, который будет использован.
- **`cooldown`** (`table` | `nil`) — информация о времени восстановления:
  - `durationMs` (`number`) — длительность кулдауна в миллисекундах.
  - `remainingMs` (`number`) — оставшееся время кулдауна в миллисекундах.
- **`pointInfo`** (`table` | `nil`) — информация о точке установки:
  - `range` (`number`) — максимальное расстояние до цели (0, если не указано).
  - `minRange` (`number`) — минимальное расстояние до цели (0, если не указано).
  - `radius` (`number`) — радиус действия AoE от точки применения (0, если не указано).
- **`sysNameAEMark`** (`string` | `nil`) — имя текстуры AEMark для визуализации AoE-заклинаний.

Возможные значения поля `sysType`:
- `ENUM_CONTEXT_ACTION_TYPE_UNKNOWN` — тип не определен.
- `ENUM_CONTEXT_ACTION_TYPE_SPELL` — запуск заклинания.
- `ENUM_CONTEXT_ACTION_TYPE_EXPLOIT` — использование устройства.
- `ENUM_CONTEXT_ACTION_TYPE_LOOT` — сбор добычи.
- `ENUM_CONTEXT_ACTION_TYPE_NPC_TALK` — разговор с NPC.
- `ENUM_CONTEXT_ACTION_TYPE_ITEM` — использование предмета из инвентаря.

## Примеры
### Проверка доступности контекстного действия
```lua
local actionInfo = avatar.GetContextActionInfo( id )
if actionInfo then
  local enabled = actionInfo.enabled
  if enabled then
    -- Действие доступно для использования
  else
    -- Действие присутствует в списке, но недоступно (например, цель вне зоны действия)
  end
end
```
::: info Описание примера
В примере запрашивается информация о контекстном действии по его идентификатору. Если действие существует, проверяется значение поля `enabled`, определяющее возможность его применения в текущий момент. В зависимости от результата выполняются соответствующие ветви логики.
:::
