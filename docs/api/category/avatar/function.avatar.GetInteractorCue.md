# avatar.GetInteractorCue
Возвращает текущую реплику NPC-собеседника.

## Описание
```lua
avatar.GetInteractorCue(): table | nil
```
Извлекает и возвращает информацию о текущей реплике NPC-собеседника в диалоге. Ответы на данную реплику доступны через функцию `avatar.GetInteractorNextCues`.

::: warning Замечание
Вызов функции необходимо производить после того, как событие `EVENT_INTERACTION_STARTED` будет передано клиенту. Во время ведения диалога смена цели взаимодействия не допускается.
:::

::: tip Совет
Функция обладает базовой реализацией в игровой логике и расширенной реализацией в пользовательском интерфейсе (UI). Поля `image` и `titlePicture` в возвращаемых данных относятся исключительно к UI-части.
:::

## Список параметров
Параметры отсутствуют.

## Возвращаемые значения
Возвращает `table` или `nil`.
Если реплика отсутствует, возвращается `nil`. В противном случае возвращается таблица со следующими полями:
- **`name`** (`WString`)
Имя реплики или ответ на неё.
- **`text`** (`WString`)
Текстовое содержимое реплики.
- **`data`** (`table`)
Индексированный с нуля список дополнительных данных (`CueData`).
- **`isSpecial`** (`boolean`)
Индикатор того, является ли реплика не просто текстом в разговоре, а началом специфического действия (например, показ респекта, вход в парикмахерскую и т.п.).

Элементы таблицы `data` (`CueData`) содержат следующие поля:
- **`alias`** (`string` | `nil`)
Псевдоним элемента данных. Может использоваться для подстановки этого элемента в форматную строку, заданную в поле `name` реплики.
- **`image`** (`TextureId` | `nil`)
Идентификатор текстуры для иконки реплики, если таковая имеется (реализовано только для UI).
- **`titlePicture`** (`TextureId` | `nil`)
Идентификатор текстуры для титульной картинки в тексте реплики, если таковая имеется (реализовано только для UI).

Также в `CueData` присутствует набор взаимоисключающих полей, определяющих требования для ответа:
- **`money`** (`number` | `nil`)
Количество денег, требуемых для ответа.
- **`number`** (`number` | `nil`)
Числовое значение, требуемое для ответа.
- **`itemId`** (`ObjectId` | `nil`)
Идентификатор предмета, требуемого для ответа.
- **`string`** (`WString` | `nil`)
Текстовое значение, требуемое для ответа.

## Примеры
### Получение реплики и форматирование текста с подстановкой данных
```lua
local cue = avatar.GetInteractorCue()

if cue then
    local valuedText = common.CreateValuedText()
    valuedText:SetFormat( cue.name )

    if cue.data[ 0 ] and cue.data[ 0 ].itemId then
        local itemInfo = avatar.GetItemInfo( cue.data[ 0 ].itemId )
        if itemInfo then
            valuedText:SetVal( cue.data[ 0 ].alias, itemInfo.name )
        end
    end
end
```
::: info Описание примера
Извлекается текущая реплика NPC. Если реплика существует, создается объект форматированного текста. В случае, если в нулевом индексе массива `data` содержится идентификатор предмета (`itemId`), запрашивается информация об этом предмете. При успешном получении информации, имя предмета подставляется в форматную строку по псевдониму (`alias`).
:::
