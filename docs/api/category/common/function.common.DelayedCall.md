# common.DelayedCall
Планирует выполнение указанной функции с заданной временной задержкой.

## Описание
```lua
common.DelayedCall( delayMs: number, callback: function, ...: any ): number
```
Функция планирует выполнение указанной функции с заданной задержкой в миллисекундах. Возвращает числовой идентификатор, который используется для отмены запланированного вызова через функцию `common.CancelDelayedCall`.

::: warning Замечание
Фактическая задержка выполнения может превышать заданное значение на время до двух кадров рендеринга.
:::

## Список параметров
- **`delayMs`** (`number`)
Задержка вызова в миллисекундах. Значение должно быть строго больше нуля.
- **`callback`** (`function`)
Функция, которая будет выполнена по истечении заданного времени.
- **`...`** (`any`)
От 0 до 10 произвольных аргументов, передаваемых в вызываемую функцию. Захват внешних переменных (upvalues/closure) происходит в момент вызова данной функции API.

## Возвращаемые значения
Возвращает `number` — числовой идентификатор (ссылку) запланированного вызова, используемый для его отмены через `common.CancelDelayedCall`.

## Примеры
### Уничтожение виджета после завершения анимации
```lua
wtPopUp:PlayFadeEffect( nil, 0, 750 )
common.DelayedCall( 800, wtPopUp.DestroyWidget, wtPopUp )
```
::: info Описание примера
Инициируется воспроизведение эффекта плавного изменения прозрачности виджета длительностью 750 миллисекунд. Параллельно планируется вызов метода уничтожения данного виджета с задержкой 800 миллисекунд, что гарантирует удаление элемента интерфейса после полного завершения визуального эффекта.
:::
