# common.GetAllodsGoals
Возвращает список текущих целей аватара.

## Описание
```lua
common.GetAllodsGoals(): table
```
Функция возвращает таблицу со списком целей аватара, структура данных которых соответствует интерфейсу `AllodsGoalInterface`. Информация становится доступной для получения только после успешного завершения процесса входа в игру.

::: warning Замечание
При изменении состояния существующих целей или появлении новых присылается событие `EVENT_ALLODS_GOAL_CHANGED`.
:::

## Список параметров
Параметры отсутствуют.

## Возвращаемые значения
Возвращает `table` — список, индексация которого начинается с 0, содержащий информацию о целях аватара. Каждый элемент списка представляет собой таблицу со следующей структурой:
- **`goalId`** (`ObjectId`)
Идентификатор цели (GoalId).
- **`state`** (`number` | `integer`)
Состояние цели (1 — активна, 2 — выполнена).
- **`locator`** (`table` | `nil`)
Локатор для функции автобега. Если локатор отсутствует, возвращается `nil`. В противном случае таблица содержит поля:
  - `zoneId` (`ObjectId`) — идентификатор интерфейсной карты.
  - `position` (`GamePosition`) — координаты точки.
- **`stages`** (`table` | `nil`)
Список, индексация которого начинается с 0, содержащий информацию о подцелях аватара. Если подцели отсутствуют, возвращается `nil`. Структура элементов списка идентична основной цели и включает поля `goalId`, `state` и `locator`.

## Примеры
### Получение и перебор списка целей аватара
```lua
local goals = common.GetAllodsGoals()

if goals then
    for i = 0, GetTableSize(goals) - 1 do
        local goal = goals[i]
        LogInfo("Цель: ", goal.goalId, " Состояние: ", goal.state)

        if goal.stages then
            for j = 0, GetTableSize(goal.stages) - 1 do
                local stage = goal.stages[j]
                LogInfo("  Подцель: ", stage.goalId, " Состояние: ", stage.state)
            end
        end
    end
end
```
::: info Описание примера
В примере запрашивается таблица целей аватара. При успешном получении данных происходит перебор списка с помощью цикла, начиная с индекса 0. Для каждой цели выводится её идентификатор и состояние. Дополнительно проверяется наличие подцелей (`stages`), и при их наличии выполняется вложенный перебор для вывода информации о каждой подцели.
:::
