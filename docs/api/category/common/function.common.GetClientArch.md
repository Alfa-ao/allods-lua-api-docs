# common.GetClientArch
Возвращает архитектуру клиента игры.

## Описание
```lua
common.GetClientArch(): number
```
Функция возвращает числовое значение, представляющее архитектуру клиента игры. Полученное значение используется для определения среды выполнения и выбора соответствующих бинарных модулей или вариантов кода.

## Список параметров
Параметры отсутствуют.

## Возвращаемые значения
Возвращает `number` — архитектуру клиента. Значение соответствует одной из констант перечисления `CLIENT_ARCH_*`:
- **`CLIENT_ARCH_WIN32`** — 32-разрядная архитектура Windows.
- **`CLIENT_ARCH_WIN64`** — 64-разрядная архитектура Windows.

## Примеры
### Выбор и загрузка специфичного для архитектуры кода
```lua
local code = codeVariants[ common.GetClientArch() ]
if code then
  loadstring( code )()
else
    LogError( "No supported chunk variant found" )
end
```
::: info Описание примера
В примере из таблицы `codeVariants` извлекается вариант кода, соответствующий текущей архитектуре клиента, полученной через `common.GetClientArch()`. Если код найден, он компилируется и выполняется с помощью `loadstring`. Если поддержка для текущей архитектуры отсутствует, в лог выводится сообщение об ошибке.
:::
