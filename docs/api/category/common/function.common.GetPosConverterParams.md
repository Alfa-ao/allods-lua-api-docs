# common.GetPosConverterParams
Возвращает параметры окна игры.

## Описание
```lua
common.GetPosConverterParams(): table
```
Функция возвращает таблицу с параметрами окна игры, описывающими текущие и референсные размеры в виртуальной и реальной системах координат.

В игре существуют две системы координат экрана: виртуальная и реальная. Обе начинают отсчёт с нуля от верхнего левого угла клиентской области окна (или экрана в полноэкранном режиме). Обе используются для указания местоположений и размеров виджетов.
- **Виртуальные единицы** существуют специально для позиционирования виджетов и используются наиболее часто. По ним позиционируются виджеты с выравниванием `WIDGET_ALIGN_LOW`, `WIDGET_ALIGN_HIGH`, `WIDGET_ALIGN_CENTER`, `WIDGET_ALIGN_BOTH`.
- **Реальные (абсолютные) единицы** используются крайне редко, только для виджетов, позиционируемых по `WIDGET_ALIGN_LOW_ABS`.

Текущий виртуальный размер окна по одной из осей равен базовому (референсному) размеру, а по другой варьируется в зависимости от пропорций окна. Например, если пропорции окна шире чем 16:9, виртуальная высота равна референсной, а ширина рассчитывается относительно высоты и превышает референсную. Если пропорции уже 16:9, виртуальная ширина равна референсной, а высота рассчитывается относительно ширины. Виртуальный размер экрана никогда не может быть меньше референсного.

Реальный размер окна представляет собой размер в реальных пикселях операционной системы.

Референсный размер окна задается в настройках игры (Масштаб интерфейса) и хранится в `global.cfg`. Базовое значение — 1920x1080.
Коэффициенты масштабирования (на них домножается референсное разрешение) для разных значений настройки:
- Минимальный — 1.33333 (2560x1440)
- Мелкий — 1.22222
- Уменьшенный — 1.11111
- Стандартный — 1 (1920x1080)
- Увеличенный — 0.94815 (1820x1024)
- Крупный — 0.88888
- Максимальный — 0.83333 (1600x900)

::: warning Замечание
Можно задать произвольное значение референсного разрешения через `global.cfg`, однако это не рекомендуется:
```ini
gfx_reference_custom = 1
gfx_reference_size_x = [800:3840]
gfx_reference_size_y = [600:2160]
```
Корректность отображения на нестандартных референсных разрешениях не тестируется.
:::

## Список параметров
Параметры отсутствуют.

## Возвращаемые значения
Возвращает `table` со следующими полями:
- **`fullVirtualSizeX`** (`number`) — текущий виртуальный размер окна по оси X.
- **`fullVirtualSizeY`** (`number`) — текущий виртуальный размер окна по оси Y.
- **`realSizeX`** (`number`) — реальный размер окна по оси X.
- **`realSizeY`** (`number`) — реальный размер окна по оси Y.
- **`referenceVirtualSizeX`** (`number`) — базовый виртуальный размер окна по оси X.
- **`referenceVirtualSizeY`** (`number`) — базовый виртуальный размер окна по оси Y.

## Примеры
### Получение и вывод параметров окна игры
```lua
local posConverter = common.GetPosConverterParams()

LogInfo( "fullVirtualSizeX: ", posConverter.fullVirtualSizeX )
LogInfo( "fullVirtualSizeY: ", posConverter.fullVirtualSizeY )
LogInfo( "realSizeX: ", posConverter.realSizeX )
LogInfo( "realSizeY: ", posConverter.realSizeY )
LogInfo( "referenceVirtualSizeX: ", posConverter.referenceVirtualSizeX )
LogInfo( "referenceVirtualSizeY: ", posConverter.referenceVirtualSizeY )
```
::: info Описание примера
В примере запрашиваются параметры окна игры через `common.GetPosConverterParams()`. Затем в информационный лог выводятся текущие виртуальные, реальные и референсные размеры окна по осям X и Y.
:::

При разрешении экрана 1920x1080 в полноэкранном режиме и стандартных настройках масштаба интерфейса результат выполнения примера будет следующим:
```text
fullVirtualSizeX: 1920
fullVirtualSizeY: 1080
realSizeX: 1920
realSizeY: 1080
referenceVirtualSizeX: 1920
referenceVirtualSizeY: 1080
```
