# customizationLib.GetActiveSlotInfo
Возвращает информацию о кастомизируемом (активном) слоте в состоянии кастомизации.

## Описание
```lua
customizationLib.GetActiveSlotInfo(): table|nil
```
Функция возвращает таблицу с информацией о кастомизируемом (активном) слоте в состоянии кастомизации или `nil`, если главный игрок не настраивает слот. Возвращаемая таблица содержит числовой и строковый идентификаторы слота, а также идентификатор размещенного предмета. Если кастомизация слота происходит в личном приключении, таблица дополнительно включает числовой и строковый идентификаторы типа монстра.

## Список параметров
Параметры отсутствуют.

## Возвращаемые значения
Возвращает `table` или `nil`.

Если главный игрок настраивает слот, возвращается таблица со следующими полями:
- **`slot`** (`number`) - Номер слота в общем пространстве множества слотов, используемых во всей системе. Можно использовать для хранения состояния, но точное множество значений в общем случае не известно.

- **`sysSlot`** (`string`) - Строковое обозначение слота.
- **`itemId`** (`ObjectId` | `nil`) - Идентификатор предмета, если он есть в слоте.
- **`beastType`** (`number`) - Тип монстра. Поле присутствует, если настраивается слот в личном приключении.
- **`sysBeastType`** (`string`) - Строковое обозначение типа монстра. Поле присутствует, если настраивается слот в личном приключении.

::: details Допустимые значения
Для полей `slot` и `sysSlot` используется перечисление `ENUM_SlotName_...`.
Для полей `beastType` и `sysBeastType` используется перечисление `ENUM_BeastType_...`.
:::

## Примеры
### Проверка состояния кастомизации и получение информации о слоте
```lua
if customizationLib.IsActive() then
    local info = customizationLib.GetActiveSlotInfo()
end
```
::: info Описание примера
В примере проверяется состояние кастомизации с помощью функции `customizationLib.IsActive()`. Если процесс активен, извлекается информация о текущем настраиваемом слоте посредством вызова `customizationLib.GetActiveSlotInfo()`.
:::
