# hangar.GetShipInfo

Возвращает информацию по кораблю, находящемуся в ангаре.

## Описание

```lua
hangar.GetShipInfo( shipId: ObjectId ): table|nil
```

Функция извлекает и возвращает информацию по кораблю, находящемуся в ангаре.

::: warning Замечание
Функция имеет основную реализацию в логике и расширенную реализацию в UI (система UIState/GameState).
:::

## Список параметров

- **`shipId`** (`ObjectId`) - Идентификатор корабля.

## Возвращаемые значения

Возвращает `table` или `nil`.

Если корабль найден, возвращается таблица со следующими полями:

- **`id`** (`ObjectId`) - Идентификатор корабля.

- **`name`** (`WString`) - Имя корабля такого типа.

- **`sysDebugName`** (`string` | `nil`) - Дебажный идентификатор ресурса или `nil` для финальной версии.

- **`markers`** (`table` | `nil`) - `nil`, если нет информации; иначе набор маркеров, описывающих состояние корабля. Каждый отдельный маркер, которым помечен корабль, представлен элементом таблицы с ключом типа `string`(`ENUM_ShipInfoMarker_*`) и значением `true`.

- **`image`** (`TextureId`) - Идентификатор иконки корабля такого типа (реализовано только для UI).

- **`isSummoned`** (`boolean`) - `true`, если корабль был уже вызван.

- **`mass`** (`number`) - Масса корабля без устройств.

## Примеры

### Получение информации о кораблях в ангаре

```lua
local ships = hangar.GetShips()
if ships then
    for i, shipId in pairs( ships ) do
        local shipInfo = hangar.GetShipInfo( shipId )
        if shipInfo then
            local name = shipInfo.name
        end
    end
end
```

::: info Описание примера
В примере извлекается список идентификаторов кораблей в ангаре. Далее в цикле для каждого корабля запрашивается подробная информация. При успешном получении данных из таблицы извлекается имя корабля.
:::
