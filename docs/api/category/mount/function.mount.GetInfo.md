# mount.GetInfo
Возвращает информацию о ездовом животном главного игрока.

## Описание
```lua
mount.GetInfo( mountId: ObjectId ): table | nil
```
Функция извлекает и возвращает детальную информацию о ездовом животном по его уникальному идентификатору. Возвращаемая таблица содержит исчерпывающие данные о характеристиках, состоянии, опыте и параметрах взаимодействия с животным. Если животное с указанным идентификатором не найдено или отсутствует, функция возвращает `nil`.

## Список параметров
- **`mountId`** (`ObjectId`)
Уникальный идентификатор ездового животного.

## Возвращаемые значения
Возвращает `table` или `nil`.
Если животное найдено, возвращается таблица со следующими полями:
- **`id`** (`ObjectId`) — уникальный идентификатор животного.
- **`grade`** (`number`) — грейд животного.
- **`name`** (`WString`) — название животного.
- **`description`** (`ValuedText` | `nil`) — описание животного.
- **`sysName`** (`string` | `nil`) — служебное название.
- **`debugName`** (`string` | `nil`) — путь к файлу описания предмета (отладочная информация).
- **`canGainExperience`** (`boolean`) — получает ли животное опыт будучи призванным и прокачивает ли свой уровень.
- **`experience`** (`number`) — опыт, накопленный животным.
- **`level`** (`number`) — уровень, зависящий от накопленного опыта.
- **`levelExperience`** (`number`) — базовый опыт для текущего уровня.
- **`nextLevelExperience`** (`number`) — опыт, необходимый для следующего уровня.
- **`satiationMs`** (`number`) — текущее время до полного истощения в миллисекундах (изменяется при кормлении).
- **`canBeFeeded`** (`boolean`) — допускается ли кормление данного животного в принципе.
- **`canBeMorphed`** (`boolean`) — допускается ли применение в сочетании с метаморфом.
- **`isProbe`** (`boolean`) — является ли данный маунт пробником.
- **`isExoMount`** (`boolean`) — является ли данный маунт экзоскелетом.
- **`experienceToday`** (`number`) — количество опыта, полученного за текущие сутки.
- **`timeActivatedLeftMs`** (`number` | `nil`) — количество миллисекунд до исчезновения животного из конюшни, если у животного есть время жизни (время уменьшается, только если животное вызвано в данный момент).
- **`timeTotalLeftMs`** (`number` | `nil`) — количество миллисекунд до исчезновения животного из конюшни, если у животного есть время жизни (время уменьшается в любом случае).
- **`currentLevelStats`** (`table`) — таблица с полями, описывающими характеристики маунта на текущем уровне:
  - `speed` (`number`) — скорость (м/с).
  - `health` (`number`) — максимум здоровья.
  - `regen` (`number`) — регенерация (количество восстановленного здоровья в секунду).
  - `castTimeMs` (`number`) — время для вызова из конюшни в миллисекундах.
- **`nextLevelStats`** (`table`) — таблица с полями, описывающими характеристики маунта на следующем уровне (структура полей аналогична `currentLevelStats`).
- **`hasLender`** (`boolean`) — `true`, если ездовое животное выдано лучшим другом.
- **`lenderId`** (`ObjectId` | `nil`) — идентификатор лучшего друга, который дал ездовое животное.
- **`hasBorrower`** (`boolean`) — `true`, если ездовое животное выдано лучшему другу.
- **`borrowerId`** (`ObjectId` | `nil`) — идентификатор лучшего друга, которому ездовое животное отдано.
- **`image`** (`TextureId`) — идентификатор текстуры для иконки маунта (реализовано исключительно для UI).

## Примеры
### Получение характеристик здоровья маунта
```lua
local mountInfo = mount.GetInfo( mount.GetMounts()[0] )
if mountInfo then
    local currentHealth = mountInfo.currentLevelStats.health
    local nextHealth = mountInfo.nextLevelStats.health
end
```
::: info Описание примера
В примере запрашивается информация о первом доступном ездовом животном из списка, полученного через `mount.GetMounts()`. Если данные успешно извлечены, извлекаются значения максимального здоровья на текущем и следующем уровнях из вложенных таблиц `currentLevelStats` и `nextLevelStats`.
:::
