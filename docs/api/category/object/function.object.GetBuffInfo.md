# object.GetBuffInfo

Возвращает информацию о бафе или его ресурсе.

## Описание

```lua
object.GetBuffInfo( id: ObjectId | BuffId, includeResourceInfo: boolean | nil, onlyResourceInfo: boolean | nil ): table | nil
```

Функция извлекает и возвращает детальную информацию о бафе. В качестве идентификатора может выступать как ID экземпляра бафа, так и ID ресурса бафа. Набор информации для ресурса бафа является подмножеством набора информации для экземпляра бафа.

## Список параметров

- **`id`** (`ObjectId` | `BuffId`) - Идентификатор экземпляра (должен быть валидным) или ресурса бафа.

- **`includeResourceInfo`** (`boolean` | `nil`) - Включать ли информацию о ресурсе бафа при получении информации о экземпляре бафа. По умолчанию `true`.

- **`onlyResourceInfo`** (`boolean` | `nil`) - Включать только информацию о ресурсе бафа при получении информации о экземпляре бафа. По умолчанию `false`.

## Возвращаемые значения

Возвращает `table` | `nil` - информация о бафе или `nil` при ошибке. Таблица содержит следующие поля:

- **`buffId`** (`BuffId`) - ID ресурса бафа. Доступно для ресурса и экземпляра.

- **`name`** (`WString`) - имя бафа. Доступно для ресурса и экземпляра.

- **`sysName`** (`string`) - внутреннее имя бафа, соответствующее поле в базе для бафа - `sysUIScriptName`. Доступно для ресурса и экземпляра.

- **`texture`** (`UITextureId`) - идентификатор текстуры для иконки бафа. Доступно для ресурса и экземпляра.

- **`isStackable`** (`boolean`) - стекается баф или нет. Доступно для ресурса и экземпляра.

- **`stackLimit`** (`number`) - максимальное количество бафов данного типа. Доступно для ресурса и экземпляра.

- **`isPositive`** (`boolean`) - `true`, если баф позитивный. Доступно для ресурса и экземпляра.

- **`isGradual`** (`boolean`) - `true`, если стэки бафа спадают по одному. Доступно для ресурса и экземпляра.

- **`canDetach`** (`boolean`) - можно ли скликивать такой баф. Доступно для ресурса и экземпляра.

- **`isNeedVisualizeDuration`** (`boolean`) - нужно ли показывать duration. Доступно для ресурса и экземпляра.

- **`priority`** (`number`(`ENUM_UIBuffPriority_*`)) - приоритет отображения бафа в интерфейсе. Доступно для ресурса и экземпляра.

- **`groups`** (`table`) - таблица групп бафа. Ключ - системное имя группы, значение - `true`. Безымянные группы не учитываются. Доступно для ресурса и экземпляра.

- **`gainSpells`** (`table`) - список заклинаний, которые усиливает данный баф; таблица индексированная с 1. Доступно для ресурса и экземпляра.

- **`isNeedVisualize`** (`boolean`) - нужно ли показывать этот баф в панели бафов. Если `false` для ресурса, то `false` для всех экземпляров. Обратное неверно. Доступно для ресурса и экземпляра, но реализация отличается.

- **`id`** (`ObjectId`) - ID объекта бафа. Доступно только для экземпляра бафа.

- **`ownerId`** (`ObjectId` | `nil`) - ID объекта, на котором висит баф, если такой объект есть. Доступно только для экземпляра бафа.

- **`casterId`** (`ObjectId` | `nil`) - идентификатор кастера, повесившего баф, если его можно получить. Доступно только для экземпляра бафа.

- **`stackCount`** (`number`) - может быть количеством одинаковых бафов, уровнем бафа или аналогичной информацией. Доступно только для экземпляра бафа.

- **`durationMs`** (`number`) - полное время действия бафа. Доступно только для экземпляра бафа.

- **`remainingMs`** (`number`) - время до окончания действия бафа. Доступно только для экземпляра бафа.

- **`debugName`** (`string`) - отладочная информация (путь до файла). Только для DEVELOPER_CLIENT.

- **`debugGroups`** (`table`) - отладочная информация (путь до файла группы). Только для DEVELOPER_CLIENT.

## Примеры

### Обработка события добавления бафа

```lua
function buffManager:OnAddBuff( eventData )
    if not self.trackedObjects[ eventData.objectId ] then
        return
    end
    if not self.resourceCache[ eventData.resourceId ] then
        self.resourceCache[ eventData.resourceId ] = object.GetBuffInfo( eventData.resourceId )
    end
    self:AddBuff( eventData.objectId, object.GetBuffInfo( eventData.buffId, false ) )
end

function buffManager:GetActiveEventHandlers()
    return {
        { self.OnAddBuff, "EVENT_OBJECT_BUFF_ADDED", { isNeedVisualize = true }, { cfilter = true } }
    }
end
```

::: info Описание примера
В примере реализуется обработка события добавления бафа. Сначала проверяется наличие объекта в списке отслеживаемых. Затем кэшируется информация о ресурсе бафа, если она еще не была получена. После этого добавляется сам баф с получением его информации без данных о ресурсе. Также демонстрируется регистрация обработчика события с фильтрами.
:::
